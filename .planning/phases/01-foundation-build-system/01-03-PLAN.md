<task type="auto">
  <name>Task 1: Create scripts/validate.sh (G0-G3 validation)</name>
  <files>scripts/validate.sh</files>
  <action>Create `scripts/validate.sh` implementing validation gates G0-G3. This is BUILD-01 in action.

The script MUST follow the pattern established in `scripts/validate-vm.sh`:
- `#!/usr/bin/env bash` shebang (not `#!/bin/bash`)
- `set -euo pipefail` on line 3
- One-line description comment on line 2

Implementation requirements per RESEARCH.md code example (lines 483-536):

1. **`gate()` helper function:** Takes gate_id, description, and command. Runs the command, prints "GATE {id}: PASS/FAIL -- {description}", increments PASS/FAIL counters.

2. **G0 -- Repo hygiene:**
   - Required files exist: README.md, AGENTS.md, .gitignore
   - No ISO files committed: `find . -name "*.iso" -not -path "./.git/*"`
   - No `.env` files committed (except .env.example): `find . -name ".env" -not -name ".env.example" -not -path "./.git/*"`

3. **G1 -- Profile integrity:**
   - `profiles/aetheros/profiledef.sh` exists
   - `profiles/aetheros/packages.x86_64` exists
   - `profiles/aetheros/pacman.conf` exists

4. **G2 -- Package policy:**
   - No duplicate packages in packages.x86_64 (use `sort | uniq -d`, filter comments with `grep -v '^#'`)
   - packages.x86_64 is not empty

5. **G3 -- Script validation:**
   - For each `scripts/*.sh` file, run `shellcheck -x "$script"`
   - If shellcheck not installed, print WARN and skip (do not fail)

6. **Output format:**
   - Header with date and git SHA
   - Per-gate PASS/FAIL lines
   - Summary: "=== Results: N passed, M failed ==="
   - Exit 0 if FAIL==0, exit 1 otherwise

CRITICAL: Per the critical rules, use `grep -v '^#' | grep -c` for counting non-comment lines, not bare `grep -c` which counts comments.

Make the script executable: `chmod +x scripts/validate.sh`

Follow the shared pattern from PATTERNS.md: REPO_ROOT resolution using SCRIPT_DIR for all repo-relative paths.</action>
  <verify>
    <automated>chmod +x scripts/validate.sh &amp;&amp; shellcheck -x scripts/validate.sh &amp;&amp; ./scripts/validate.sh 2&gt;&amp;1 | grep -q 'GATE'</automated>
  </verify>
  <done>scripts/validate.sh is executable, passes shellcheck, runs G0-G3 gates and produces PASS/FAIL output per gate. Exits 0 on a properly configured repo.</done>
</task>
