<task type="auto">
  <name>Task 1: Create print-build-env.sh and build-iso.sh</name>
  <files>scripts/print-build-env.sh, scripts/build-iso.sh</files>
  <action>Create two scripts: print-build-env.sh (Stage 0 environment capture) and build-iso.sh (G4 ISO build).

Step 1 — Create `scripts/print-build-env.sh`:
- Standard boilerplate: `#!/usr/bin/env bash`, description comment, `set -euo pipefail`
- Echo the following build environment variables with `$(command)` substitution and fallbacks:
  - Timestamp: `date -u +%Y-%m-%dT%H:%M:%SZ`
  - Git SHA: `git rev-parse --short HEAD` (fallback: 'N/A')
  - Branch: `git rev-parse --abbrev-ref HEAD` (fallback: 'N/A')
  - Host OS: `uname -s -r -m`
  - Kernel: `uname -v`
  - archiso version: `pacman -Q archiso | awk '{print $2}'` (fallback: 'NOT INSTALLED')
  - mkarchiso path: `command -v mkarchiso` (fallback: 'NOT FOUND')
  - shellcheck version: `shellcheck --version | grep 'version:' | awk '{print $2}'` (fallback: 'NOT INSTALLED')
  - QEMU version: `qemu-system-x86_64 --version | head -1` (fallback: 'NOT INSTALLED')
  - Git version: `git --version | awk '{print $3}'` (fallback: 'NOT INSTALLED')
- Per RESEARCH.md code example (lines 678-695) for the exact format
- Make executable: `chmod +x scripts/print-build-env.sh`

Step 2 — Create `scripts/build-iso.sh`:
- Standard boilerplate
- REPO_ROOT resolution: `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` / `REPO_ROOT="$(cd "${SCRIPT_DIR}/.." && pwd)"`
- Set PROFILE, WORK, OUT paths relative to REPO_ROOT
- Dependency check: `for cmd in mkarchiso; do command -v "$cmd" || echo "ERROR: ${cmd} not found. Install with: sudo pacman -Syu --needed archiso" >&2; exit 1; done`
- Profile validation: `test -f "${PROFILE}/profiledef.sh"` or exit 1
- Clean previous work dir: `rm -rf "${WORK}"` if exists
- Create out dir: `mkdir -p "${OUT}"`
- Run: `mkarchiso -v -w "${WORK}" -o "${OUT}" "${PROFILE}"`
- Post-build: `ls -lh "${OUT}"/*.iso` or warn
- Per RESEARCH.md code example (lines 540-583)
- Make executable: `chmod +x scripts/build-iso.sh`

Per Pitfall 3: mkarchiso is not installed on the build host. The script must detect this and fail with a clear install command message, NOT silently fail.
Per Pitfall 5: profiledef.sh format must be correct. The script validates it exists before invoking mkarchiso.</action>

<task type="auto">
  <name>Task 2: Create smoke-qemu.sh and collect-artifacts.sh</name>
  <files>scripts/smoke-qemu.sh, scripts/collect-artifacts.sh</files>
  <action>Create two scripts: smoke-qemu.sh (G6 VM boot smoke test) and collect-artifacts.sh (G5 artifact capture).

Step 1 — Create `scripts/smoke-qemu.sh`:
- Standard boilerboilerplate
- Argument validation: `ISO_PATH="${1:?Usage: $0 <path-to-iso>}"`
- File check: `test -f "${ISO_PATH}"` or exit 1 with error
- Tool preference with fallback:
  - If `command -v run_archiso`: `run_archiso -i "${ISO_PATH}"`
  - Else: `qemu-system-x86_64 -enable-kvm -m 2048 -smp 2 -cdrom "${ISO_PATH}" -boot d -vga virtio -display sdl "$@"`
- Per RESEARCH.md code example (lines 588-616)
- Make executable: `chmod +x scripts/smoke-qemu.sh`

Note: This script boots a GUI VM. It will wait for QEMU to exit. This is expected -- the user interacts with the VM and closes it.

Step 2 — Create `scripts/collect-artifacts.sh`:
- Standard boilerplate
- REPO_ROOT resolution
- Set VERSION="0.1.0-demo.1", ARTIFACTS path
- Find ISO: `ISO_FILE=$(find "${OUT}" -maxdepth 1 -name "*.iso" | head -1)` with error if not found
- Create artifacts dir: `mkdir -p "${ARTIFACTS}"`
- SHA256: `(cd "${OUT}" && sha256sum *.iso > "${ARTIFACTS}/SHA256SUMS")`
- Build env: heredoc writing VERSION, GIT_SHA, BUILD_DATE, HOST_OS, ARCHISO_VERSION to build-env.txt
- Package list: `cp profiles/aetheros/packages.x86_64 artifacts/v0.1.0-demo.1/package-list.txt`
- Manifest: Write MANIFEST.md with version, date, git SHA, ISO name, SHA256, package count
- Per RESEARCH.md code example (lines 620-674)
- Make executable: `chmod +x scripts/collect-artifacts.sh`

Per Pitfall 4: pacman keyring issues. The collect-artifacts.sh script does not need to handle keyring; build-iso.sh should document that users may need `pacman-key --init && pacman-key --populate archlinux` if signature verification fails.</action>
