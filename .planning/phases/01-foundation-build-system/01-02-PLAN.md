<objective>
Create the mkarchiso profile `profiles/aetheros/` -- the single source of truth for the ISO (D-04). This profile defines what packages are installed, how the ISO is named, and how it boots. Everything downstream (build scripts, validation, ISO creation) depends on this profile existing and being valid.

Purpose: The profile is the foundation of the reproducible build. Without it, mkarchiso has nothing to build from.
Output: profiledef.sh, packages.x86_64, pacman.conf, and boot config directories (syslinux, grub, efiboot).
</objective>

  <name>Task 1: Create profiledef.sh and packages.x86_64</name>
  <files>profiles/aetheros/profiledef.sh, profiles/aetheros/packages.x86_64</files>
  <action>Create the two core profile files for the aetheros mkarchiso profile (D-03, D-04).

Step 1 — Create the profile directory:
```bash
mkdir -p profiles/aetheros
```

Step 2 — Create `profiles/aetheros/profiledef.sh` with the following content. Per RESEARCH.md Pattern 1 and the archiso README.profile.rst, these are the required variables. The `install_dir` must be max 8 chars, `[a-z0-9]` only.

```bash
#!/usr/bin/env bash
# profiles/aetheros/profiledef.sh

iso_name="aetheros"
iso_label="AETHEROS_01"
iso_publisher="aetherOS Project <https://github.com/anphuni/aetherOS>"
iso_application="aetherOS Live/Rescue CD"
iso_version="0.1.0-demo.1"
install_dir="aetheros"
buildmodes=("iso")
bootmodes=("bios.syslinux" "uefi.grub")
arch="x86_64"
```

Step 3 — Create `profiles/aetheros/packages.x86_64` with the minimal bootable package list. Per RESEARCH.md Pattern 3 and the archiso README: `mkinitcpio` and `mkinitcpio-archiso` are MANDATORY. Per D-03, this is a minimal bootable profile -- just enough to reach XFCE. Per D-02 discretion area, use `linux-lts` for stability.

```
# profiles/aetheros/packages.x86_64
# Minimal bootable Arch + XFCE system
# Phase 1: just enough to reach graphical session

# Base system (required by archiso)
base
mkinitcpio
mkinitcpio-archiso

# Kernel
linux-lts

# Xorg
xorg-server
xorg-xinit

# Desktop
xfce4
xfce4-goodies

# Display manager
lightdm
lightdm-gtk-greeter

# Networking
networkmanager

# Filesystem tools
dosfstools
e2fsprogs
```

Key constraints:
- One package per line, no versions
- Comments use `#`
- No Debian package names (no apt-utils, firefox-esr)
- Must include mkinitcpio and mkinitcpio-archiso (archiso README mandatory requirement)
- Must include base, linux-lts, xorg-server, xfce4, lightdm

Per RESEARCH.md Pitfall 6: omitting mkinitcpio or mkinitcpio-archiso will cause mkarchiso to fail.</action>

No Debian package names present.</done>

<task type="auto">
  <name>Task 2: Create pacman.conf and copy boot configuration files</name>
  <files>profiles/aetheros/pacman.conf, profiles/aetheros/syslinux/, profiles/aetheros/grub/, profiles/aetheros/efiboot/</files>
  <action>Create the pacman.conf and copy boot configuration files from the Arch releng profile. Per RESEARCH.md Pattern 1 and the "Don't Hand-Roll" table: boot configuration should be copied from the releng profile, not created from scratch.

Step 1 — Copy the releng profile's pacman.conf and boot configs:
```bash
RELENG="/usr/share/archiso/configs/releng"

# Copy pacman.conf
cp "${RELENG}/pacman.conf" profiles/aetheros/pacman.conf

# Copy boot configuration directories
cp -r "${RELENG}/syslinux" profiles/aetheros/syslinux
cp -r "${RELENG}/grub" profiles/aetheros/grub
cp -r "${RELENG}/efiboot" profiles/aetheros/efiboot
```

If the releng profile is not available at that path (e.g., archiso not yet installed), create a minimal pacman.conf manually:
```bash
cat > profiles/aetheros/pacman.conf << 'PACMAN'
[options]
HoldPkg = pacman glibc
Architecture = auto
CheckSpace
Color
ILoveCandy
SigLevel = Required DatabaseOptional
LocalFileSigLevel = Optional

[Include]
Include = /etc/pacman.d/mirrorlist

[core]
Include = /etc/pacman.d/mirrorlist

[extra]
Include = /etc/pacman.d/mirrorlist

[multilib]
Include = /etc/pacman.d/mirrorlist
PACMAN
```

And create minimal boot config directories:
```bash
mkdir -p profiles/aetheros/syslinux
mkdir -p profiles/aetheros/grub
mkdir -p profiles/aetheros/efiboot
```

Note: If boot config directories are empty, the build scripts will still function -- mkarchiso has defaults. The critical files are profiledef.sh and packages.x86_64.

Step 2 — Verify the profile structure:
```bash
find profiles/aetheros/ -type f -o -type d | sort
```

Per D-04, the profile structure follows Arch releng convention. Per RESEARCH.md recommended project structure, the profile should contain: profiledef.sh, packages.x86_64, pacman.conf, airootfs/, syslinux/, grub/, efiboot/.</action>
