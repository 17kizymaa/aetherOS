<task type="auto">
  <name>Create GRUB bootloader configuration</name>
  <files>profiles/aetheros/grub/grub.cfg</files>
  <action>
Create `profiles/aetheros/grub/grub.cfg` with the following content. This configures GRUB for UEFI boot of the aetherISO live system.

Key requirements (D-06, D-07, D-08):
- GRUB timeout: 1 second
- Default entry: first (and only) menu entry
- Kernel parameters: `quiet splash`
- Boot entry points to the Arch ISO squashfs image

Write this exact content to `profiles/aetheros/grub/grub.cfg`:

```
# profiles/aetheros/grub/grub.cfg
# GRUB bootloader config for aetherOS UEFI
# Phase 2: Core Desktop & Boot

set timeout=1
set default=0

# Menu entry for aetherOS Live
menuentry "aetherOS Live" {
    set gfxpayload=keep
    search --no-floppy --set=root --label AETHEROS_01
    linux /boot/vmlinuz-linux-lts img_loop=/aetheros/airootfs.sfs earlymodules=loop quiet splash
    initrd /boot/initramfs-linux-lts.img
}
```

Critical details:
- `search --no-floppy --set=root --label AETHEROS_01` uses the ISO label from profiledef.sh to find the boot device
- `img_loop=/aetheros/airootfs.sfs` points to the squashfs image (mkarchiso places it at `/aetheros/` inside the ISO)
- `earlymodules=loop` ensures the loop module is loaded early for squashfs
- `quiet splash` suppresses verbose boot messages (D-08)
- `set gfxpayload=keep` preserves the native resolution

Note: The exact `img_loop` path and `search` parameters follow the Archiso GRUB convention. If mkarchiso generates its own grub.cfg during build, this file in the profile directory serves as the source that mkarchiso copies into the ISO's EFI boot partition.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build host -> ISO | Build scripts run mkarchiso which assembles the ISO from untrusted package sources (Arch repos). Package integrity relies on pacman signature verification. |
| GRUB config -> Kernel | GRUB config passes kernel parameters. Malicious config could alter boot behavior. Config is committed to repo (trusted source). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | packages.x86_64 | mitigate | Validation gate G2 checks for duplicates and policy compliance. Only Arch official repo packages. |
| T-02-02 | Tampering | grub/grub.cfg | mitigate | Config is committed to repo, reviewed before build. No runtime modification. |
| T-02-03 | Information Disclosure | profiledef.sh | accept | ISO metadata is public information. No secrets in profile. |
| T-02-SC | Tampering | pacman installs during build | mitigate | pacman package signatures verified by keyring. Build host must have initialized keyring (pitfall 6). |
</threat_model>
