**Part B: Demo user creation service**

The archiso base system does not create a 'demo' user by default. We need a systemd service that creates the user at first boot if it doesn't already exist.

Create `profiles/aetheros/airootfs/etc/systemd/system/create-demo-user.service` with this exact content:

```ini
# airootfs/etc/systemd/system/create-demo-user.service
# Create demo user at first boot (one-shot)
# Phase 2: Core Desktop & Boot

[Unit]
Description=Create demo user for aetherOS
Before=lightdm.service
ConditionFirstBoot=yes

[Service]
Type=oneshot
ExecStart=/usr/bin/bash -c '\
  if ! id demo &>/dev/null; then \
    useradd -m -G wheel -s /bin/bash demo; \
    echo "demo" | passwd --stdin demo 2>/dev/null || true; \
  fi'
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
```

Key details:
- `Before=lightdm.service` -- ensures user exists before LightDM tries to auto-login
- `ConditionFirstBoot=yes` -- only runs on first boot (idempotent)
- Creates user with home directory (`-m`), adds to `wheel` group (`-G wheel`)
- Sets password to "demo" (for cases where auto-login fails and manual login is needed)
- `RemainAfterExit=yes` -- service shows as active after execution

Enable this service by creating a symlink:
```bash
mkdir -p profiles/aetheros/airootfs/etc/systemd/system/multi-user.target.wants
ln -sf ../create-demo-user.service profiles/aetheros/airootfs/etc/systemd/system/multi-user.target.wants/create-demo-user.service
```

**Part A: zram swap configuration** (D-04, D-05)

Create `profiles/aetheros/airootfs/etc/systemd/zram-generator.conf` with this exact content:

```ini
# airootfs/etc/systemd/zram-generator.conf
# zram swap for aetherOS (1 GB, zstd compression)
# Phase 2: Core Desktop & Boot

[zram0]
zram-size = 1024
compression-algorithm = zstd
```

Key details:
- `zram-size = 1024` -- 1 MB? No, zram-generator uses MiB by default. 1024 MiB = 1 GiB. (D-04: 1 GB zram swap)
- `compression-algorithm = zstd` -- best compression/speed tradeoff for low-RAM systems (D-04)
- The `zram-generator` package (added to packages.x86_64 in Plan 01) reads this config at boot

**Part C: Polkit rules for GUI shutdown/reboot** (D-03)

Create `profiles/aetheros/airootfs/etc/polkit-1/rules.d/10-shutdown-reboot.rules` with this exact content:

```javascript
// airootfs/etc/polkit-1/rules.d/10-shutdown-reboot.rules
// Allow demo user to shutdown/reboot without password
// Phase 2: Core Desktop & Boot

polkit.addRule(function(action, subject) {
    if ((action.id == "org.freedesktop.login1.power-off" ||
         action.id == "org.freedesktop.login1.reboot" ||
         action.id == "org.freedesktop.login1.power-off-multiple-sessions" ||
         action.id == "org.freedesktop.login1.reboot-multiple-sessions") &&
        subject.user == "demo") {
        return polkit.Result.YES;
    }
});
```

Key details:
- Allows both `power-off` and `reboot` actions for the `demo` user
- Includes `multiple-sessions` variants for completeness
- `.rules` extension required for polkit to pick it up
- Numeric prefix `10-` for ordering
