# aetherOS v0.1 Operating Contract

aetherOS v0.1 is not yet an independent operating system. It is a reproducible workflow layer built on a stable Linux base.

The goal of this sprint is to demonstrate that old or low-end hardware can become a calm, useful, local-first production environment for independents, artists, and small businesses.

Rules:

1. Debian Stable + XFCE is the default base for this sprint.
2. No custom kernel, package manager, programming language, or independent repository.
3. All product changes must be represented as scripts, manifests, configs, docs, or validation logs.
4. Chat logs are not source of truth. The repository is source of truth.
5. Agents may propose changes, but humans approve merges.
6. Every build must have a fallback path.
7. The demo must prove one workflow: create, organize, protect, and resume real work.
8. AI features are optional accelerators, not hard dependencies.
9. Licensing and provenance must be recorded from the beginning.
10. If a feature does not improve the handoff demo, it is deferred.

Product copy

# aetherOS

aetherOS turns old and low-end computers into calm local production workstations.

Built for independents, artists, and small businesses, it combines a lightweight Linux base with opinionated project workflows, backup/recovery tools, local documentation, and optional AI assistance.

The first promise is simple:

Your machine should help you finish work, protect your files, and stay usable — without forcing you into cloud lock-in or hardware churn.

Final note

Do not build “an OS” this week. Build a repeatable old-machine production workflow that boots, guides, backs up, and feels inevitable. That is aetherOS v0. Everything else can earn its way in later.
